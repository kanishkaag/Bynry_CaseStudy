# Bynry Backend Case Study

This repository contains my solution for the Backend Engineering Intern case study assigned by Bynry.

The case study focuses on building core components of a B2B inventory management system, including debugging an existing API, designing a scalable database schema, and implementing a low-stock alert endpoint.

The solution demonstrates my approach to:

* Identifying and fixing real-world backend issues
* Designing relational database structures with proper constraints
* Implementing efficient and scalable API endpoints
* Handling assumptions and incomplete requirements

All tasks have been implemented with a focus on clean code, clarity, and practical backend design considerations.
